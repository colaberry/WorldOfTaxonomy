[CV] CVE/CWE Types -- The CV entry groups the most common vulnerability categories defined by the Common Weakness Enumeration (CWE) that are frequently referenced in Common Vulnerabilities and Exposures (CVE) reports. It provides a concise mapping of CVE identifiers to their primary CWE types, helping analysts quickly understand the nature of each security flaw. This top-level classification aids prioritization and remediation planning.
[CV.01] CWE-79: Cross-Site Scripting (XSS) -- CV.01 designates CWE-79, Cross-Site Scripting (XSS), a weakness that lets attackers inject untrusted data into a web page so it runs in a victim's browser, enabling data theft, session hijacking or content defacement. It is one of the most common web-application vulnerabilities listed in the CWE Top 25.
[CV.02] CWE-89: SQL Injection -- CV.02 identifies CWE-89, a SQL injection flaw where untrusted input is embedded in database queries without proper sanitization, allowing an attacker to alter or execute arbitrary SQL commands. This can lead to data disclosure, modification, or complete system compromise.
[CV.03] CWE-787: Out-of-bounds Write -- CV.03 corresponds to CWE-787, an out-of-bounds write vulnerability where software writes data beyond the intended memory buffer, potentially overwriting adjacent data, causing crashes or allowing code execution. This defect is a common class of memory-corruption bugs used in many exploits.
[CV.04] CWE-20: Improper Input Validation -- CWE-20 defines improper input validation as a weakness where software fails to correctly verify data before processing it, allowing malformed or malicious input to affect behavior. This can lead to security issues such as injection, buffer overflows, or logic errors, making it a common root cause in many CVE disclosures.
[CV.05] CWE-125: Out-of-bounds Read -- CV.05 corresponds to CWE-125, an out-of-bounds read where software accesses memory beyond the intended buffer limits. This can expose sensitive data, cause crashes, or facilitate further attacks because the program fails to enforce proper bounds checking on input or internal pointers.

[CV.07] CWE-416: Use After Free -- CV.07 classifies the CWE-416 "Use After Free" vulnerability, a memory-management error where software accesses a heap or stack object after it has been released. The dangling pointer can cause crashes, data corruption, or allow an attacker to execute arbitrary code by manipulating the reclaimed memory. This category is part of the CWE Top list used to group common software weaknesses.
[CV.08] CWE-22: Path Traversal -- CWE-22, Path Traversal, describes flaws that let an attacker supply crafted file-path input to move up a directory hierarchy and access files or directories outside the intended location. This can expose sensitive data, overwrite files, or execute code, and is a common issue in web and file-handling applications.
[CV.09] CWE-352: CSRF -- CV.09 maps to CWE-352, Cross-Site Request Forgery (CSRF), a web-application flaw where an attacker tricks a victim's browser into submitting unauthorized requests that inherit the user's credentials. The exploit relies on the victim's active session and can cause unwanted actions such as fund transfers or data changes. Defenses include anti-CSRF tokens, same-site cookies, and verifying request origins.
[CV.10] CWE-434: Unrestricted File Upload -- CVE vulnerability type CV.10 refers to CWE-434, Unrestricted File Upload, where an application accepts files from users without enforcing sufficient validation on type, size, or content. This allows attackers to upload malicious payloads that can be executed, accessed, or cause unintended behavior on the server.
[CV.11] CWE-862: Missing Authorization -- CWE-862, labeled Missing Authorization, describes weaknesses where a system fails to verify that a user is permitted to perform a requested action, allowing any authenticated or unauthenticated user to access functions or data beyond their rights. This omission can lead to privilege escalation, data exposure, or unauthorized operations.
[CV.12] CWE-476: NULL Pointer Dereference -- CWE-476, listed as CV.12 in the CVE vulnerability type categories, denotes a null pointer dereference. It occurs when software attempts to read or write through a pointer that has not been initialized and holds a null value, often causing crashes, denial-of-service conditions, or providing an avenue for attackers to execute arbitrary code.
[CV.13] CWE-190: Integer Overflow -- CV.13 corresponds to CWE-190, which identifies integer overflow flaws where calculations exceed the maximum value a variable can hold, causing wrap-around or truncation. Such errors can corrupt data, bypass security checks, or enable arbitrary code execution.
[CV.14] CWE-502: Deserialization -- CWE-502: Deserialization refers to weaknesses that arise when an application reconstructs objects or data structures from untrusted input without proper validation, allowing attackers to manipulate the serialized content to execute malicious code, alter program flow, or cause denial-of-service conditions. This category is among the most frequently observed CWE top-rated vulnerability types.
[CV.15] CWE-287: Improper Authentication -- CVE entry CV.15 refers to CWE-287 Improper Authentication, which occurs when a product fails to verify the identity of users, devices, or other systems before granting access. This weakness can allow attackers to bypass login controls, gain unauthorized privileges, or conduct session hijacking.
