[SL] SPDX License Categories -- SL is the code for the SPDX License Categories section, which groups the SPDX identifier list of open-source licenses into broad types such as permissive, copyleft, and proprietary. It helps organizations classify and manage software compliance by providing a standardized taxonomy for license analysis.
[SL.01] Permissive: MIT -- SL.01 denotes the MIT license, a permissive open-source license that grants users broad rights to use, copy, modify, and distribute the software with minimal restrictions. It requires only attribution of the original copyright notice and includes a disclaimer of warranties.
[SL.02] Permissive: Apache-2.0 -- SL.02 denotes the Apache License 2.0, a permissive open-source license in the SPDX Software License Identifier taxonomy. It grants users the freedom to use, modify, and distribute the software, requires preservation of copyright notices and a copy of the license, and includes an express patent-grant clause while limiting liability and warranty.
[SL.03] Permissive: BSD-2-Clause -- SL.03 identifies the BSD-2-Clause license as a permissive open-source license. It permits redistribution and use of source or binary forms provided the original copyright notice and disclaimer are retained, and it does not impose copyleft requirements or restrictions on derived works.
[SL.04] Permissive: BSD-3-Clause -- SL.04 refers to the BSD-3-Clause license, a permissive open-source license that allows redistribution and modification of source code with few conditions. It requires preservation of the original copyright notice, disclaimer of liability, and a statement that the names of the project or its contributors may not be used to endorse derived products. This license imposes minimal restrictions, encouraging broad reuse in both open-source and proprietary software.

[SL.06] Copyleft: GPL-2.0 -- SL.06 denotes the GNU General Public License version 2.0, a strong copyleft license. It permits users to run, study, modify, and redistribute the software, but any distributed derivative works must be licensed under the same GPL-2.0 terms, ensuring that the source remains freely available.
[SL.07] Copyleft: GPL-3.0 -- SL.07 identifies the GNU General Public License version 3.0, a strong copyleft license that obliges any modified or combined works to be distributed under the same GPL-3.0 terms, ensuring that source code remains freely available to downstream users.
[SL.08] Weak Copyleft: LGPL-2.1 -- SL.08 denotes the GNU Lesser General Public License version 2.1, a weak-copyleft license that permits linking the licensed code with proprietary software while requiring that any modifications to the LGPL-covered components be released under the same license, ensuring continued freedom for the original work.
[SL.09] Weak Copyleft: LGPL-3.0 -- SL.09 denotes a weak-copyleft license, specifically the GNU Lesser General Public License version 3.0. It permits linking the licensed library with proprietary software while requiring that any modifications to the library itself be distributed under the same LGPL-3.0 terms. This balances open-source sharing of the library with flexibility for downstream developers.
[SL.10] Weak Copyleft: MPL-2.0 -- SL.10 denotes a weak-copyleft license, specifically the Mozilla Public License version 2.0. MPL-2.0 permits combining the licensed code with proprietary software, but any modifications to the MPL-covered files must be made available under the same license, ensuring source-code transparency for those parts.
[SL.11] Network Copyleft: AGPL-3.0 -- SL.11 denotes the Network Copyleft category in the SPDX license taxonomy. It groups licenses that extend copyleft obligations to users who interact with the software over a network, the most common example being the GNU Affero General Public License version 3.0.

[SL.14] Public Domain: CC0-1.0 -- SL.14 denotes the Creative Commons Zero 1.0 Universal public domain dedication, a license that waives all copyright and related rights, placing the work in the public domain worldwide. It allows anyone to use, modify, and distribute the work without restriction or attribution.
[SL.15] Public Domain: Unlicense -- The Unlicense is a public-domain dedication that explicitly waives all copyright and related rights, permitting anyone to use, modify, and distribute the work without any conditions, including attribution. It places the software in the public domain worldwide, and where a dedication is not recognized it functions as a permissive, no-restriction license. This entry (SL.15) categorises the Unlicense within the SPDX software license identifier set.
[SL.16] Proprietary/Commercial -- SL.16 denotes a proprietary or commercial software license, meaning the code is not openly licensed and the licensor imposes restrictions on use, modification, and distribution. This classification is used in SPDX to identify licenses that are closed-source and typically require payment or a specific agreement for usage.
