[HS] HTTP Status Code Classes -- HS identifies the five primary classes of HTTP response status codes, distinguished by their leading digit: 1xx informational, 2xx success, 3xx redirection, 4xx client error, and 5xx server error.
[HS.1xx] 1xx Informational -- HS.1xx groups the HTTP 1xx informational status codes, which signal that a request has been received and that processing is continuing without a final response. Examples include 100 Continue, 101 Switching Protocols, and 102 Processing; these responses are interim and typically contain no message body.
[HS.200] 200 OK -- HS.200 denotes the standard HTTP response indicating that the request has succeeded. The server has fulfilled the client's request and the requested resource is transmitted in the response body, if applicable. It is the most common success status in web communication.
[HS.201] 201 Created -- HS.201 denotes the HTTP 201 Created status, indicating that a request has been fulfilled and a new resource has been created as a result. The response usually includes a Location header that points to the newly created resource and may contain a representation of that resource.
[HS.2xx] 2xx Success -- The HS.2xx category denotes HTTP response codes ranging from 200 to 299, indicating that a client's request was received, understood, and successfully processed by the server. These status codes confirm the requested action was carried out and typically include responses such as "OK," "Created," and "No Content."
[HS.301] 301 Moved Permanently -- HS.301 indicates the requested resource has been permanently moved to a new URL. Clients should update bookmarks and use the provided Location header for future requests. Search engines transfer ranking to the new address.
[HS.302] 302 Found -- The 302 Found status signals that the requested resource is temporarily located at a different URI. The client should follow the location supplied in the response but continue to use the original URI for subsequent requests.
[HS.3xx] 3xx Redirection -- HS.3xx denotes the class of HTTP response status codes indicating redirection. These codes tell the client that further action is needed to complete the request, typically by following a different URI. Common examples include 301 Moved Permanently and 302 Found.
[HS.400] 400 Bad Request -- HS.400 denotes the HTTP 400 Bad Request status, indicating that the server cannot process the request because of malformed syntax, invalid framing, or deceptive routing. It signals a client-side error, prompting the client to correct the request before retrying.
[HS.401] 401 Unauthorized -- The 401 Unauthorized status signals that the requested resource requires authentication and the client has not provided valid credentials. The server returns this code when authentication is missing or fails, prompting the client to resend the request with appropriate authentication information.
[HS.403] 403 Forbidden -- HS.403 denotes the HTTP 403 Forbidden status, indicating that the server understood the request but refuses to authorize it. It is used when authentication is provided but insufficient privileges, and the client should not repeat the request without modification.
[HS.404] 404 Not Found -- The 404 Not Found status indicates that the server was reachable but could not locate the requested resource, often because the URL is incorrect or the content has been removed. It signals a client-side error, prompting the user to verify the address or try another link.

[HS.500] 500 Internal Server Error -- HS.500 denotes the 500 Internal Server Error category in HTTP response status codes. It signals that the server encountered an unexpected condition that prevented it from fulfilling the client's request, often indicating a generic failure on the server side.
[HS.502] 502 Bad Gateway -- HS.502 denotes the HTTP 502 Bad Gateway status, indicating that a server acting as a gateway or proxy received an invalid response from an upstream server. It signals a temporary communication failure between servers and typically prompts the client to retry the request.
[HS.503] 503 Service Unavailable -- HS.503 denotes the HTTP 503 Service Unavailable status, indicating that the server cannot handle the request at the moment, typically because it is overloaded or down for maintenance, and may provide a Retry-After header suggesting when to try again.
[HS.5xx] 5xx Server Error -- HS.5xx designates the 5xx Server Error category of HTTP response status codes, indicating that the server failed to fulfill a valid request due to an internal condition. These codes signal that the problem lies with the server, not the client, and include responses such as 500 Internal Server Error, 502 Bad Gateway, and 504 Gateway Timeout. Use this classification to group and track server-side error conditions in web applications.
