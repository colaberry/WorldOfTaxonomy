[W3C] W3C Standard Groups -- The W3C Standard Groups are the working groups within the World Wide Web Consortium
[W3C.A11Y] Accessibility (WCAG, WAI-ARIA) -- W3C.A11Y refers to the web accessibility specifications maintained by the World Wide Web Consortium, including the Web Content Accessibility Guidelines and the WAI-ARIA suite. These standards define techniques to make web content perceivable, operable, understandable and robust for people with disabilities.
[W3C.API] Web APIs -- Web APIs are standardized interfaces defined by the W3C that allow web applications to interact with browsers, devices, and online services. They provide a consistent set of methods and events for tasks such as manipulating the DOM, handling media, and accessing hardware features, ensuring cross-platform compatibility.
[W3C.DID] Decentralized Identifiers -- W3C.DID is the specification for Decentralized Identifiers, a new type of globally unique identifier that can be created, owned, and controlled by the subject without reliance on a central authority, enabling peer-to-peer trust, verifiable credentials, and interoperable identity across web services.
[W3C.DOM] DOM and Events -- The W3C.DOM entry defines the Document Object Model and its event handling mechanisms as standardized interfaces for accessing and manipulating HTML and XML documents.
[W3C.HTML] HTML and CSS -- The W3C.HTML entry covers the core markup language HTML and its companion stylesheet language CSS, defining the syntax, document structure, and presentation rules that enable interoperable web pages across browsers. It references W3C specifications and best practices for accessibility, responsive design, and modern web development.
[W3C.PAY] Web Payments -- W3C.PAY designates the W3C Web Payments specification, which defines a set of browser-based APIs and protocols allowing merchants to request and process payments securely and consistently across devices. It includes the Payment Request API that streamlines user interaction, supports multiple payment methods, and promotes interoperability between browsers, payment providers, and e-commerce platforms.
[W3C.PER] Web Performance -- W3C.PER, titled "Web Performance," defines the World Wide Web Consortium's standards for measuring and improving the speed and efficiency of web resources. It includes guidelines for load-time metrics, resource prioritisation, and optimization techniques to ensure fast, responsive user experiences across browsers and network conditions.
[W3C.PWA] Progressive Web Apps -- Progressive Web Apps (PWA) are web applications that use modern browser capabilities to deliver app-like experiences, including offline functionality, push notifications, and home-screen installation. They combine responsive design with service workers and web app manifests to provide fast, reliable, and engaging interactions across devices.
[W3C.SEC] Web Security -- W3C.SEC is the W3C Web Standards entry that defines the organization's security guidelines for web technologies. It covers recommended practices for authentication, data integrity, privacy, and secure communication across browsers and web services, aiming to help developers implement robust protection against common threats.
[W3C.SEM] Semantic Web (RDF, OWL) -- W3C.SEM designates the Semantic Web standards suite, encompassing technologies like RDF (Resource Description Framework) for data modeling and OWL (Web Ontology Language) for defining rich ontologies. These specifications enable machines to interpret, interlink, and reason about web content, supporting more intelligent data integration and automated inference across distributed sources.
[W3C.SVG] Graphics (SVG, Canvas) -- W3C.SVG defines the W3C's specifications for Scalable Vector Graphics and related Canvas APIs, providing a markup language and scripting model for resolution-independent, interactive 2D graphics in browsers. The standards ensure cross-platform rendering, CSS styling, and seamless integration with HTML and the DOM, enabling rich visual content on the web.
[W3C.VC] Verifiable Credentials -- Verifiable Credentials (VC) is a W3C specification that defines a data model and cryptographic mechanisms for issuing, presenting, and verifying digital credentials in a tamper-evident, privacy-preserving way. It lets individuals and organizations receive credentials such as diplomas or licenses from issuers, store them securely, and share proof of claims with relying parties without exposing unnecessary data.
[W3C.WEB] WebRTC and Communications -- W3C.WEB - WebRTC and Communications is the W3C specification set that defines real-time communication APIs for peer-to-peer audio, video, and data transfer directly in browsers. It details protocol standards, security measures, and interoperability guidelines to enable seamless media streaming and data channels without the need for plugins.
[W3C.WS] Web Services -- W3C.WS, titled Web Services, is the W3C's suite of standards that enables interoperable machine-to-machine communication over the web, defining protocols such as SOAP, description languages like WSDL, and guidelines for RESTful services to ensure consistent, platform-independent interaction.
[W3C.XML] XML Technologies -- XML is a W3C-defined markup language that encodes data in a hierarchical, text-based format readable by both humans and machines, allowing custom tags and extensibility. It serves as the foundation for many web-service protocols and related standards such as XSLT, XPath, and SOAP.
