[BT] Bluetooth SIG Profile Types -- BT designates the set of Bluetooth SIG Profile Types, which are standardized specifications defining the behavior and data formats for particular Bluetooth applications. These profiles ensure interoperability between devices by outlining required services, command sets, and performance criteria for functions such as audio streaming, health monitoring, or device discovery. They are maintained by the Bluetooth Special Interest Group and referenced in product certification.
[BT.01] Advanced Audio Distribution (A2DP) -- The Advanced Audio Distribution Profile (A2DP) defines how high-quality stereo audio is streamed over Bluetooth, allowing devices such as phones, tablets, headphones and speakers to exchange audio data and basic playback controls wirelessly. It supports selectable codecs, sample rates and optional metadata to maintain sound fidelity during transmission.
[BT.02] Audio/Video Remote Control (AVRCP) -- BT.02 defines the Audio/Video Remote Control Profile, which enables a Bluetooth device such as a smartphone to control playback functions of another device like a speaker or TV. It supports commands for play, pause, track navigation, volume and metadata exchange, allowing interoperable remote control across audio/video products.
[BT.03] Basic Imaging (BIP) -- BT.03 identifies the Basic Imaging profile defined by the Bluetooth SIG. The profile enables devices to exchange images and related metadata over Bluetooth, supporting functions such as image push, pull, and printing. It provides standardized commands for image transfer between phones, cameras, printers, and other peripherals.
[BT.04] Basic Printing (BPP) -- Basic Printing (BPP) is a Bluetooth SIG profile that lets devices send print jobs to a printer over a Bluetooth link using the OBEX protocol. It supports common document formats such as JPEG, PNG, and PDF and provides basic features like job status and queue management, making it suitable for simple, low-power printing scenarios.
[BT.05] Device ID (DID) -- BT.05 defines the Device ID (DID) profile, a standard Bluetooth service that lets a device advertise its vendor, product, and version information to other devices. It enables automatic identification and driver selection for peripherals such as keyboards, mice, and game controllers, simplifying setup and improving interoperability across the Bluetooth ecosystem.

[BT.07] Health Device (HDP) -- The Health Device Profile (HDP) defines a standardized framework for Bluetooth-enabled medical and wellness devices to exchange data
[BT.08] Human Interface Device (HID) -- The Human Interface Device (HID) profile defines how Bluetooth devices such as keyboards, mice, game controllers and other input peripherals communicate with host devices, enabling standardized low-latency exchange of input reports and control commands. It supports both boot and report protocols and allows for secure, power-efficient operation across various operating systems.
[BT.09] Message Access (MAP) -- The Bluetooth Message Access Profile (MAP) enables devices to exchange and manage text messages such as SMS, MMS or emails over a Bluetooth link. It provides functions for notification, message browsing, retrieval and sending, allowing head-units, wearables and other accessories to access a phone's messaging store in a standardized way.
[BT.10] Object Push (OPP) -- Object Push (OPP) is a Bluetooth profile that enables simple file exchange between devices, allowing users to send images, contacts, or other data objects with minimal setup. It operates over the OBEX protocol and is commonly used for quick transfers without requiring a full file-system browsing capability.
[BT.11] Personal Area Network (PAN) -- BT.11 specifies the Personal Area Network (PAN) profile, allowing Bluetooth devices to create a small network for data exchange and internet sharing. It defines roles such as Network Access Point, Group Network, and User, enabling IP-based communication between phones, laptops, printers and other peripherals.
[BT.12] Phone Book Access (PBAP) -- BT.12 designates the Phone Book Access Profile, a Bluetooth protocol that lets devices such as car kits or wearables retrieve contacts and recent call logs from a mobile phone. It defines the services, data formats and transport mechanisms used for secure phone-book exchange over RFCOMM.
[BT.13] Serial Port (SPP) -- The Serial Port Profile (SPP) defines how Bluetooth devices emulate a classic RS-232 serial cable, allowing legacy applications to exchange data over a wireless link. It uses the RFCOMM protocol as transport and supports point-to-point and point-to-multipoint connections, making it common in devices such as printers, GPS receivers, and medical equipment.
[BT.14] Generic Attribute (GATT) -- BT.14 Generic Attribute (GATT) defines how Bluetooth Low Energy devices discover services, read and write characteristics, and receive notifications using the Attribute Protocol, providing a common framework for exposing data and control functions across devices.
[BT.15] Mesh Profile -- The Bluetooth SIG Mesh Profile (BT.15) defines the specifications for creating many-to-many networks over Bluetooth Low Energy, allowing devices to relay messages and form robust, self-healing mesh topologies. It covers provisioning, security, message handling, and configuration models so that a wide range of IoT applications can communicate reliably across large areas.
[BT.16] LE Audio (BAP/TMAP) -- BT.16 designates the LE Audio profile, which includes the Basic Audio Profile (BAP) for high-quality, low-power audio streaming and the Telephony and Media Audio Profile (TMAP) for coordinated multi-device audio handling in Bluetooth Low Energy environments.
