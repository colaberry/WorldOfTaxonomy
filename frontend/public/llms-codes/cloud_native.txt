[CN] CNCF Landscape -- CN is the top-level entry for the CNCF Landscape, a visual map that organizes all Cloud Native Computing Foundation projects and related tools into categories such as orchestration, networking, security and observability, offering a high-level overview of the cloud-native ecosystem.
[CN.01] App Definition and Development -- CN.01 - App Definition and Development groups the CNCF projects that help specify, build, package, and deliver cloud-native applications. It includes frameworks, SDKs, build tools, CI/CD pipelines, and language-specific libraries that enable developers to define application requirements, compile source code, create container images, and automate deployment workflows across diverse environments.
[CN.02] Orchestration and Management -- CN.02 denotes the Orchestration and Management category in the CNCF Cloud Native Landscape. It encompasses tools and platforms that automate deployment, scaling, and ongoing operation of containerized workloads, such as Kubernetes, service meshes, and CI/CD pipelines. The category helps users locate solutions that coordinate resources and ensure reliable, efficient application delivery.
[CN.03] Runtime -- The Runtime category groups software that supplies the execution environment for cloud-native workloads, such as container orchestrators, serverless platforms and the underlying compute infrastructure. These components schedule, launch and monitor containers or functions, providing the reliability, scalability and lifecycle management needed for modern applications.
[CN.04] Provisioning -- CN.04 Provisioning covers tools and services that automate the allocation, configuration, and lifecycle management of compute, storage, networking, and other infrastructure resources in cloud-native environments. It includes solutions for infrastructure as code, cluster creation, and resource scaling that enable consistent, repeatable deployment of platforms and workloads.
[CN.05] Observability and Analysis -- The Observability and Analysis category (CN.05) gathers cloud-native tools that generate, collect, store, and visualize telemetry such as metrics, logs, and traces to monitor system health, diagnose issues, and inform performance optimization. It includes monitoring platforms, logging frameworks, tracing systems, and analytics dashboards that enable operators to gain insight into distributed applications.
[CN.06] Serverless -- CN.06 - Serverless is a cloud-native execution model where developers deploy functions without managing servers, and the provider automatically handles scaling, patching and infrastructure. It enables event-driven code execution and includes platforms such as Knative, OpenFaaS and AWS Lambda, forming a distinct category in the CNCF Cloud Native Landscape.
[CN.07] Service Mesh -- Service mesh is a dedicated infrastructure layer that manages service-to-service communication in cloud-native environments, providing traffic routing, load balancing, security, and observability without modifying application code. It abstracts networking concerns, enabling developers to focus on business logic while the mesh handles resilience and policy enforcement. In the CNCF Cloud Native Landscape, this capability is identified by the code CN.07.
[CN.08] API Gateway -- API Gateways are network-level services that expose, route, and protect application APIs, handling request transformation, authentication, rate limiting, and observability. They act as a single entry point for client traffic, enabling consistent security policies and traffic management across microservices. Typical implementations include Kong, Ambassador, and Traefik.
[CN.09] Container Registry -- The Container Registry (code CN.09) in the CNCF Cloud Native Landscape groups services that store, manage and distribute container images, offering versioning, vulnerability scanning and access control for developers and CI/CD pipelines. It covers both open-source and commercial registries that integrate with Kubernetes and other cloud-native tools.
[CN.10] Security and Compliance -- CN.10 groups projects that protect cloud-native workloads and ensure they meet regulatory and internal standards. It includes solutions for vulnerability scanning, runtime protection, policy enforcement, secret management, and audit reporting, helping operators secure containers, Kubernetes clusters and the supply chain while maintaining compliance documentation.
[CN.11] Database (Cloud Native) -- CN.11 in the CNCF Cloud Native Landscape identifies the Database (Cloud Native) category, covering data stores that are designed to run as containerized, orchestrated services. It includes distributed, fault-tolerant databases built for dynamic scaling, automated management, and integration with cloud-native ecosystems such as Kubernetes.
[CN.12] Streaming and Messaging -- The Streaming and Messaging category in the CNCF Cloud Native Landscape groups tools that enable real-time data pipelines, event-driven architectures, and asynchronous communication between services. It includes distributed log systems, message brokers, and stream processing platforms that provide reliable, scalable, and low-latency data transport for cloud-native applications.
[CN.13] Continuous Integration/Delivery -- CN.13 Continuous Integration/Delivery covers tools and platforms that automate building, testing and deploying code in cloud-native environments, enabling rapid and reliable software changes. The category includes CI servers, pipeline orchestrators and related services that integrate with containers and Kubernetes.
[CN.14] GitOps -- GitOps is a set of practices that uses Git repositories as the single source of truth for declarative infrastructure and application configurations, enabling automated reconciliation and continuous delivery through pull-request workflows. It streamlines deployments, improves auditability, and integrates tightly with cloud-native tooling.
