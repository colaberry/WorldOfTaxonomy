[WC] WCAG Principles -- The WC entry summarizes the four foundational Web Content Accessibility Guidelines principles-Perceivable, Operable, Understandable and Robust-that define how web content must be designed so people with diverse abilities can perceive, navigate, comprehend and interact with it reliably across current and future technologies.
[WC.01] Perceivable: Text Alternatives -- WC.01 requires that all non-text content provide a text alternative that conveys the same information, function, or purpose, allowing users who rely on screen readers or other assistive technologies to perceive the content.
[WC.02] Perceivable: Time-Based Media -- WC.02 covers the Perceivable principle for time-based media in the Web Content Accessibility Guidelines. It mandates that audio and video content offer captions, transcripts, audio descriptions, and proper synchronization so users with visual or hearing impairments can perceive the information.
[WC.03] Perceivable: Adaptable -- WC.03 requires that web content be adaptable so that it can be presented in multiple formats without losing meaning, allowing users to customize the layout, orientation, or visual presentation to suit their needs. This includes using semantic markup, proper headings, and providing mechanisms such as responsive design or alternative text that enable assistive technologies and user agents to restructure information appropriately.
[WC.04] Perceivable: Distinguishable -- WC.04 focuses on the "Distinguishable" principle of WCAG, requiring that information and UI components be presented so users can perceive them easily. It mandates sufficient contrast between text and background, clear separation of foreground and background, and mechanisms to adjust or pause auditory content, ensuring that visual and auditory cues do not blend together and that users with low vision or hearing impairments can still access the content.
[WC.05] Operable: Keyboard Accessible -- WC.05 defines the operable principle of keyboard accessibility, requiring that every interactive element and function on a website can be accessed and used via a keyboard alone. This ensures users who cannot use a mouse, such as those with motor impairments or using assistive technologies, can navigate, operate controls, and complete tasks without barriers.
[WC.06] Operable: Enough Time -- WC.06 addresses the Operable guideline that users must be given sufficient time to read and interact with content; it requires that any time limits can be extended, paused, or disabled, and that users have control over timing adjustments to accommodate different speeds or needs.
[WC.07] Operable: Seizures and Physical Reactions -- WC.07 addresses the Operable principle by requiring web content to avoid causing seizures or other physical reactions, such as those triggered by flashing or rapidly changing visuals. It mandates that any potentially harmful animation be either eliminated, reduced in intensity, or provide users with controls to pause, stop, or hide it, ensuring safe interaction for individuals with photosensitive conditions.
[WC.08] Operable: Navigable -- WC.08 - Operable: Navigable requires that users can move through website content in a predictable and logical manner, with clear headings, consistent navigation mechanisms, and accessible links that allow efficient location of information using keyboards, assistive technology, or other input methods.
[WC.09] Operable: Input Modalities -- WC.09 requires that web content be operable through multiple input methods, ensuring users can interact without relying on a single device. It mandates providing keyboard access, touch targets, voice controls or other alternatives so that people with diverse abilities can navigate, activate controls and complete tasks.
[WC.10] Understandable: Readable -- WC.10 requires that text content be presented so readers can understand it easily, using clear language, appropriate reading level, and sufficient contrast for legibility. It mandates defining abbreviations, providing definitions for unfamiliar words, and ensuring text can be resized without loss of content or functionality.
[WC.11] Understandable: Predictable -- WC.11 mandates that web content behave in predictable ways so users can anticipate the results of their actions. Interface components must maintain consistent behavior across pages, and any automatic changes in context, such as redirects or updates, must be user-initiated or clearly indicated.
[WC.12] Understandable: Input Assistance -- WC.12 addresses input assistance, requiring web content to help users avoid and correct errors when entering data. It mandates clear instructions, error detection, and suggestions for correction, such as automatic reformatting or contextual hints, so that input tasks remain understandable and manageable for all users.
[WC.13] Robust: Compatible -- WC.13 of the Web Content Accessibility Guidelines mandates that web content be robust and compatible with current and future user agents, including browsers and assistive technologies. It requires the use of standard-compliant markup, avoidance of deprecated features, and thorough testing to ensure reliable interpretation across various platforms and devices.
[WC.14] Conformance Level A -- WC.14 defines Conformance Level A as the most fundamental tier of the Web Content Accessibility Guidelines, requiring that a site meet all Level A success criteria such as providing text alternatives for non-text content and ensuring keyboard operability. Satisfying this level indicates minimal but essential accessibility for users with disabilities.

[WC.16] Conformance Level AAA -- WC.16 defines the AAA conformance level of the Web Content Accessibility Guidelines, the most stringent tier. To achieve AAA, a website must satisfy every success criterion at Levels A and AA as well as the additional, stricter criteria specific to AAA, covering aspects such as extended text alternatives, enhanced readability, and more robust navigation support. This level represents the highest degree of accessibility compliance.
