[OW] OWASP Top 10 -- OW is the umbrella entry for the OWASP Top 10, the globally recognized list of the most critical web-application security risks. It is updated roughly every three years to reflect emerging threats and provides a baseline for developers, testers, and organizations to prioritize mitigation and secure coding practices.
[OW.01] A01: Broken Access Control -- Broken access control occurs when an application fails to enforce policies that restrict what authenticated users are allowed to do, enabling attackers to view, modify, or delete data, or to perform privileged functions. This weakness can arise from missing or misconfigured checks on URLs, APIs, or server-side files.
[OW.02] A02: Cryptographic Failures -- Cryptographic failures occur when applications do not protect data appropriately, using weak algorithms, improper key management, or missing encryption for sensitive information. These flaws expose data at rest or in transit to unauthorized access, leading to confidentiality breaches and compliance violations.
[OW.03] A03: Injection -- Injection flaws arise when untrusted data is sent to an interpreter as part of a command or query, allowing attackers to execute unintended commands or retrieve unauthorized data. Common examples include SQL, NoSQL, LDAP and OS-command injection, which can lead to data loss, corruption, or system compromise.
[OW.04] A04: Insecure Design -- Insecure Design refers to the lack of security considerations built into the architecture and design of a software system, where threat modeling, secure design patterns, and defensive controls are omitted or insufficient. This results in weaknesses that can be exploited across multiple components, making it easier for attackers to bypass protections and compromise the application.
[OW.05] A05: Security Misconfiguration -- Security misconfiguration occurs when servers, databases, frameworks, or application settings are left at insecure defaults, incomplete, or improperly hardened, allowing attackers to exploit exposed endpoints, verbose error messages, outdated components, or unnecessary services. Proper hardening, regular patching, and consistent configuration management mitigate this risk.
[OW.06] A06: Vulnerable and Outdated Components -- OW.06 highlights the danger of employing libraries, frameworks, or other software components that are outdated or have known vulnerabilities. Attackers can exploit these flaws to compromise the application, so organizations must keep an inventory, monitor for security patches, and promptly update or replace insecure components.
[OW.07] A07: Identification and Authentication Failures -- Identification and authentication failures occur when applications do not properly verify user identities or manage session credentials, allowing attackers to assume other users' accounts. Issues include weak or reused passwords, credential stuffing, insecure password-reset mechanisms, missing multi-factor authentication, and poorly protected session tokens, all of which can lead to unauthorized access and data compromise.
[OW.08] A08: Software and Data Integrity Failures -- Software and Data Integrity Failures occur when applications trust code, components, or data that can be altered or corrupted, often through insecure development pipelines, unverified third-party libraries, or inadequate runtime checks. Attackers exploit these gaps to inject malicious code, manipulate business logic, or alter critical data, compromising confidentiality, integrity and availability.
[OW.09] A09: Security Logging and Monitoring Failures -- OW.09, A09: Security Logging and Monitoring Failures, refers to the insufficient collection, analysis, or alerting of security-related events in web applications, which hampers the ability to detect, investigate, and respond to attacks. Without adequate logs and continuous monitoring, breaches can go unnoticed, prolonging exposure and increasing damage. Implementing comprehensive logging, proper retention, and real-time alerting mitigates this risk.
[OW.10] A10: Server-Side Request Forgery (SSRF) -- A10 Server-Side Request Forgery (SSRF) occurs when an application processes untrusted user input to initiate network requests from the server itself, allowing attackers to craft URLs that target internal services, cloud metadata endpoints, or external resources. This can lead to data leakage, privileged-access abuse, internal port scanning, or even remote code execution if vulnerable services are reached.
