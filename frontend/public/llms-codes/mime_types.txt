[MM] MIME Top-Level Types -- MM denotes the MIME Top-Level Types category in the IANA Media Type registry, grouping the primary type identifiers such as application, audio, image, text, video, multipart, model, font, and others that appear before the slash in a media type string. This top-level classification helps organize and standardize how Internet media are identified and handled.
[MM.01] application/* -- The MM.01 entry "application/*" designates the top-level IANA media type for generic application data, covering any binary or structured content that does not fit into more specific categories such as text, image, audio, or video. It signals that the payload is intended for processing by an application rather than for direct rendering, and it must be accompanied by a defined subtype (for example, application/json) to convey precise semantics.
[MM.02] audio/* -- audio/* is the generic IANA media type for any audio format. It serves as a wildcard subtype, allowing applications to match all audio content types such as audio/mpeg, audio/ogg, or audio/webm during content negotiation or filtering.
[MM.03] font/* -- MM.03 - font/* is the wildcard media type for any font format, covering TrueType, OpenType, WOFF, and other font files. It signals that the payload is a font resource when the specific subtype is unknown or when multiple font types may be acceptable.
[MM.04] image/* -- The code MM.04 in the IANA Media Type Categories designates the generic image media type, covering all subtypes that begin with image/. It is used to indicate that a resource contains visual data such as JPEG, PNG, SVG or other image formats.
[MM.05] message/* -- The "message/*" category is a top-level IANA media type that designates any message-format payload, covering subtypes such as email (message/rfc822), SIP (message/sip), and other protocol messages. It serves as a wildcard for all specific message/* sub-types.
[MM.06] model/* -- The "model/*" category is a top-level IANA media-type used for 3-dimensional model files, such as CAD, VRML, or glTF representations. It groups a range of vendor-specific subtypes that describe binary or textual formats for visualizing geometric data across web and software applications.
[MM.07] multipart/* -- The multipart/* media type is a generic category for messages whose body consists of multiple distinct parts, each with its own content type and optional headers. Used in email, HTTP and other protocols, it requires a boundary parameter to delimit the parts and enables mixed or alternative content within a single payload.
[MM.08] text/* -- The IANA Media Type Category MM.08 covers the wildcard text/* family, which includes any media type whose primary type is "text". It groups human-readable formats such as plain text, HTML, CSS, SGML, and other textual representations used for internet communication.
[MM.09] video/* -- The IANA Media Type Category MM.09 identifies all video media types, using the wildcard "video/*" to represent any specific video subtype such as video/mp4, video/webm, or video/avi. It serves as a generic label for streaming, playback, and processing of visual content across protocols and applications.
[MM.10] application/json -- MM.10 application/json is the IANA-registered media type for JSON (JavaScript Object Notation), a lightweight, text-based data interchange format that represents objects as attribute-value pairs and arrays. It is widely used in web APIs and configuration files for transmitting structured information in a human-readable form.
[MM.11] application/xml -- application/xml is the generic IANA media type for XML documents when a more specific type is not known. It indicates that the payload is an XML document and may include a charset parameter to specify character encoding. This type is used for generic XML data interchange across web and network applications.
[MM.12] application/pdf -- MM.12 - application/pdf designates the IANA-registered media type for Portable Document Format files. It signals that the content is a PDF document, a binary format created by Adobe for representing text, fonts, and graphics in a device-independent layout. The type is widely used for web downloads, email attachments, and document exchange.
[MM.13] application/octet-stream -- application/octet-stream is the generic binary data media type defined by IANA; it signals that the content consists of an arbitrary sequence of octets with no specific format, so user agents usually treat it as a downloadable file rather than display it. It serves as a fallback type when a more specific media type is unknown.
[MM.14] text/html -- MM.14 identifies the "text/html" media type, the canonical MIME type for HTML documents served on the web. It signals that the content consists of HyperText Markup Language and should be rendered by browsers as a web page, supporting standard HTML semantics and structure.
[MM.15] text/plain -- text/plain is the IANA-registered media type for unstructured plain text content. It indicates that the data consists of human-readable characters without any markup, typically using US-ASCII or an explicit charset parameter.
