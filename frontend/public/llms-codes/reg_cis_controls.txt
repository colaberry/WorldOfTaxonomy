By continuously monitoring this inventory, the control helps ensure only authorized assets are active, simplifies risk assessment, and supports effective response to vulnerabilities and incidents.

[ctrl_03] Control 03 - Data Protection -- Control 03 in the CIS Critical Security Controls v8 focuses on protecting data throughout its lifecycle. It requires organizations to identify and classify sensitive information, apply encryption and other safeguards for data at rest and in transit, enforce proper handling and disposal, and maintain secure backup and recovery processes to ensure confidentiality, integrity, and availability.

[ctrl_05] Control 05 - Account Management -- CIS Critical Security Controls v8 Control 05 establishes processes for creating, modifying, disabling, and deleting user and service accounts to ensure only authorized individuals have access. It requires regular review of account privileges, enforcement of least-privilege principles, and timely removal of inactive or unnecessary accounts to reduce the risk of unauthorized access.
[ctrl_06] Control 06 - Access Control Management -- Control 06 focuses on managing and enforcing access permissions for all systems, applications and data. It requires organizations to establish policies for granting, reviewing and revoking user accounts, employing least-privilege principles, and using mechanisms such as multi-factor authentication, role-based access and automated provisioning to reduce unauthorized access risks.
[ctrl_07] Control 07 - Continuous Vulnerability Management -- Control 07 of the CIS Critical Security Controls v8 directs organizations to implement continuous vulnerability management, establishing processes to regularly identify, assess, prioritize, and remediate security flaws in their assets. It emphasizes the use of automated scanning tools, timely patching, and ongoing monitoring to reduce the attack surface and maintain a resilient security posture.
[ctrl_08] Control 08 - Audit Log Management -- Control 08 of the CIS Critical Security Controls version 8 focuses on audit log management. It requires organizations to collect, protect, and retain logs from critical systems, ensure they are regularly reviewed, and use them to detect, investigate, and respond to security events.
[ctrl_09] Control 09 - Email and Web Browser Protections -- Control 09 requires the organization to implement technical and procedural safeguards for email and web browsers, including secure configuration, phishing and malicious-link detection, and the use of reputable filtering and anti-malware tools. It also calls for limiting user privileges, enforcing up-to-date patches, and monitoring for suspicious activity to reduce the risk of credential theft and drive-by attacks.
[ctrl_10] Control 10 - Malware Defenses -- Control 10 focuses on implementing and maintaining technical capabilities to detect, prevent, and respond to malicious software. Organizations should employ anti-malware tools, ensure they are regularly updated, conduct scheduled scans, and use application whitelisting and behavior monitoring to limit infections. The control also requires ongoing monitoring and timely remediation of identified malware incidents.
[ctrl_11] Control 11 - Data Recovery -- Control 11 requires organizations to establish and maintain reliable data backup and recovery processes that protect critical information from loss or corruption. It mandates regular, automated backups, secure storage of backup media, and periodic testing of restoration procedures to ensure data can be recovered quickly and accurately after an incident.
[ctrl_12] Control 12 - Network Infrastructure Management -- Control 12 requires organizations to securely manage and monitor network devices, enforce configuration baselines, maintain an inventory, apply patches, and use encrypted management protocols to protect the network infrastructure from unauthorized changes and vulnerabilities.
[ctrl_13] Control 13 - Network Monitoring and Defense -- Control 13 directs organizations to implement continuous, automated monitoring of network traffic, employing intrusion detection and prevention tools, and maintaining defensive measures through timely analysis of logs, alerts, and threat intelligence so that malicious activity can be quickly identified and mitigated.
[ctrl_14] Control 14 - Security Awareness and Skills Training -- Control 14 requires organizations to establish a continuous security awareness and skills training program for all employees and relevant stakeholders, ensuring they understand security policies, can recognize threats such as phishing, and respond appropriately. The control emphasizes regular, role-based training, measurable outcomes, and reinforcement through testing and awareness campaigns.
[ctrl_15] Control 15 - Service Provider Management -- Control 15 of the CIS Critical Security Controls focuses on managing service providers to protect organizational data and assets. It requires conducting due-diligence before engaging a provider, embedding security requirements into contracts, continuously monitoring provider performance, and terminating access when the relationship ends or requirements are not met.
[ctrl_16] Control 16 - Application Software Security -- Control 16 focuses on ensuring that application software is designed, developed, and maintained securely to reduce vulnerabilities. It requires organizations to establish secure coding practices, conduct code reviews, and apply security testing throughout the software lifecycle.
[ctrl_17] Control 17 - Incident Response Management -- Control 17 ensures that an organization develops and maintains an incident response program to detect, analyze, contain, eradicate, and recover from security events. It requires documented response procedures, regular testing of the plan, and coordination with external stakeholders to improve resilience and reduce impact.
[ctrl_18] Control 18 - Penetration Testing -- Control 18 requires organizations to plan and conduct regular penetration testing that simulates realistic attack scenarios against both internal and external assets, using qualified personnel or reputable service providers. The testing must identify exploitable vulnerabilities, assess the effectiveness of existing defenses, and generate actionable remediation recommendations, which are then tracked and validated to improve the overall security posture.
[ig_1] Implementation Group 1 (IG1) - Essential Cyber Hygiene -- Implementation Group 1 (IG1) in the CIS Critical Security Controls v8 defines a core set of essential cyber-hygiene practices aimed at organizations with limited resources or lower risk profiles. It selects the most fundamental controls-such as inventory, secure configurations, access management, and basic malware defenses-to establish a baseline level of protection against common threats. IG1 is intended to be achievable for small-to-medium enterprises and serves as a starting point for broader security programs.
[ig_2] Implementation Group 2 (IG2) - Managed Security -- Implementation Group 2 (IG2) - Managed Security is the tier of CIS Controls intended for organizations that have a defined security program, moderate resources, and a need to systematically manage cyber risk. It assumes basic policies and staff are in place and emphasizes consistent implementation, monitoring, and continuous improvement of the controls.
[ig_3] Implementation Group 3 (IG3) - Comprehensive Security -- Implementation Group 3 in the CIS Critical Security Controls Version 8 represents the most comprehensive security baseline, requiring organizations to fully implement all 20 controls with mature processes and continuous monitoring. It is intended for enterprises that face high threat exposure, have complex infrastructures, or must meet stringent regulatory or contractual obligations.
