## Post (edit [BRACKETS] before publishing)

If your team reconciles industry codes across datasets, I built something for you.

The classification mismatch problem costs data teams days every month:
US suppliers report in NAICS. EU partners use NACE. ILO standards use ISCO.
WHO uses ICD. None of them officially map to each other.

I spent two years hitting this wall, so I built World Of Taxonomy - an open-source
knowledge graph that connects 279 global classification systems.

What it does:
- Translates any industry, occupation, product, or health code to its equivalents in every other system via a single API call
- Profiles 249 countries with their official, regional, and recommended classification standards
- Provides full-text search across 570,178 codes simultaneously
- Ships with an MCP server for AI agents (Claude, GPT-4) to use as structured taxonomy tools

Systems covered include: NAICS, ISIC, NACE (and all 27 EU national adaptations),
SIC, ANZSIC, HS 2022, UNSPSC, SOC, ISCO-08, ESCO, ICD-11, LOINC, ATC, GDPR,
EU Taxonomy, SFDR, GRI, SASB, and 250+ more.

It is MIT licensed, self-hosted, and free forever. Docker Compose setup takes under
two minutes.

The most common use cases I've heard from early users:
1. Data engineering teams mapping supplier/customer data across jurisdictions
2. Compliance teams cross-referencing ESG and regulatory frameworks
3. Financial analysts bridging GICS sectors to national industry codes
4. HR platforms translating job codes between SOC (US), ESCO (EU), and ISCO (global)
5. AI developers giving language models structured industry knowledge

Demo and docs: https://worldoftaxonomy.com
GitHub (open source): https://github.com/colaberry/WorldOfTaxonomy

Happy to answer questions about specific use cases or classification families.
